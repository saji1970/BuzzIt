# Android SDK Local Setup Guide

This guide helps you set up and configure the Android SDK locally for the BuzzIt project.

## Quick Setup

Run the setup script to configure your Android SDK:

```powershell
.\setup-android-sdk.ps1
```

## Manual Setup

### 1. Install Android SDK

The Android SDK is typically installed with Android Studio. If you don't have it:

**Option A: Install Android Studio**
1. Download from: https://developer.android.com/studio
2. Install and open Android Studio
3. Go to `Tools > SDK Manager`
4. Install required SDK components

**Option B: Command Line Tools Only**
1. Download from: https://developer.android.com/studio#command-tools
2. Extract to a folder (e.g., `C:\Android\sdk`)
3. Run the setup script to configure paths

### 2. Required SDK Components

- **Platform Tools** (latest)
- **Build Tools** (30.0.3 or 34.0.0)
- **Android Platform** (API 33 or 34)
- **NDK** (17.2.4988734)

### 3. Set Environment Variables

**For Current Session (PowerShell):**
```powershell
$env:ANDROID_HOME = "C:\Users\saji\AppData\Local\Android\Sdk"
$env:ANDROID_SDK_ROOT = "C:\Users\saji\AppData\Local\Android\Sdk"
$env:PATH = "$env:ANDROID_HOME\platform-tools;$env:ANDROID_HOME\tools;$env:PATH"
```

**For Permanent Setup (System Environment Variables):**
1. Open System Properties → Environment Variables
2. Add User Variables:
   - `ANDROID_HOME` = `C:\Users\saji\AppData\Local\Android\Sdk`
   - `ANDROID_SDK_ROOT` = `C:\Users\saji\AppData\Local\Android\Sdk`
3. Edit `PATH` variable and add:
   - `%ANDROID_HOME%\platform-tools`
   - `%ANDROID_HOME%\tools`
   - `%ANDROID_HOME%\tools\bin`

### 4. Verify Installation

```powershell
# Check SDK location
adb version

# Check installed platforms
dir "$env:ANDROID_HOME\platforms"

# Check build tools
dir "$env:ANDROID_HOME\build-tools"
```

The `android/local.properties` file is automatically generated by the setup script. It contains:

```properties

```

**Note:** This file should NOT be committed to version control as it contains local paths.

## Troubleshooting

### SDK Not Found
- Run `.\setup-android-sdk.ps1` to detect and configure SDK
- Verify SDK path in `android/local.properties`
- Check environment variables are set correctly

### Build Tools Missing
- Open Android Studio → SDK Manager
- Install required build tools versions
- Or use: `sdkmanager "build-tools;30.0.3"`

### NDK Issues
- Ensure NDK 17.2.4988734 is installed
- Check `ndk.dir` in `local.properties` matches actual NDK path
- Install via SDK Manager or: `sdkmanager "ndk;17.2.4988734"`

### ADB Not Found
- Verify `platform-tools` is in PATH
- Check `$env:ANDROID_HOME\platform-tools\adb.exe` exists
- Restart terminal after setting environment variables
